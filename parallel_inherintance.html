Parallel Inherintance Hierarchies

–– Penjelasan Smell

Smell Parallel Inheritance Hierarcies ini terjadi setiap kali kamu membuat subkelas untuk suatu kelas, kamu merasa perlu membuat subkelas untuk kelas lain. Semuanya baik-baik saja selama hierarkinya tetap kecil. Namun dengan bertambahnya kelas-kelas baru, membuat perubahan menjadi semakin sulit. Bayangkan ada dua atau lebih hirarki parent-children class. Lalu kita perlu menambahkan (extends) satu class baru di salah satu hirarki, hirarki lain ikut-ikutan juga butuh untuk extends class mengikuti hirarki ini. Contoh: Abstract class A di-extends menjadi A1 dan A2. Lalu ada abstract class B di-extends menjadi B1 dan B2. Ketika A di-extends menjadi A3, B juga ikut-ikutan butuh B3.


–– Penyelesaian


Untuk menghadapi smell ini, kita perlu mempertimbangkan apakah cocok bila hirarki-hirarki ini digabung saja menjadi satu. Smell tidak perlu ditangani bila memang hirarki paralel ini disengaja untuk keperluan pemisahan konteks/tanggungjawab. Atau ketika digabung code malah menjadi lebih buruk. Contohnya Hirarki Shape2D selalu berkembang berbarengan dengan hirarki AreaInterface. Bila ada Shape2D yang baru, misalnya Circle. maka AreaInterface juga akan ikut-ikutan membuat anak baru CircleAreaCalculator.


Sehingga perlu dilakukan refactor kode untuk menghilangkan smell code Parallel Inherintance Hierarchies dengan menggunakan Extract Class yaitu memindahkan logika perhitungan area agar digabung saja ke dalam Shape2D.
